# HazeClue AI — RARD–MVES v2.2

**Hybrid Riemannian–Statistical EEG Inference System with Empirically Calibrated Mode Switching**

نظام ذكاء اصطناعي لتحليل إشارات المخ (EEG) في الوقت الحقيقي لتصنيف حالة التركيز الذهني. النظام مصمم ليعمل مع أجهزة EEG منخفضة الكثافة (14 قناة، أقطاب جافة) ويتعامل بذكاء مع تدهور جودة الإشارة.

---

## الفكرة الأساسية

المشكلة إن إشارات EEG من أجهزة EMOTIV مش مستقرة دائماً — الأقطاب الجافة بتتحرك، في artifacts من حركة العضلات، وأحياناً القنوات بتفصل. فبدل ما نعتمد على طريقة واحدة للتحليل ونتمنى إن الإشارة تبقى كويسة، النظام بيقرر في كل لحظة يستخدم أنهي طريقة بناءً على جودة الإشارة الفعلية.

```
إشارة كويسة  →  RARD (هندسة ريمانية، دقة عالية)
إشارة متوسطة →  MVES (إحصائيات، قوة تحمل)
إشارة سيئة   →  SAFE (رفض، لا تخمين)
```

## شرح كل مرحلة

### 1. تحميل البيانات (`data/`)

بنستخدم مجموعتين بيانات:

| Dataset | المصدر | العينات | التردد | البروتوكول |
|---------|--------|---------|--------|-----------|
| **STEW** | IEEE DataPort | 48 شخص | 128 Hz | راحة vs مهام متعددة |
| **Mendeley** | Mendeley Data | 30 شخص | 250 Hz | راحة (عيون مفتوحة) vs تركيز |

**قرارات مهمة:**
- **STEW**: الملفات بتتاخد زي ما هي — `_lo.txt` = راحة، `_hi.txt` = شغل
- **Mendeley**: الدقيقة الأخيرة (عيون مغلقة) **بتترمي تماماً** عشان الـ Alpha peaks بتلخبط الموديل. بناخد بس الدقيقة 0 (راحة) + الدقيقة 1 (تركيز)
- **Mendeley** بيتعمله downsample من 250 Hz لـ 128 Hz عشان يتوافق مع STEW

كل البيانات بتتقطع لنوافذ 4 ثواني (512 عينة) مع 50% overlap.

### 2. المعالجة المبدئية (`preprocessing/`)

**`bandpass.py`** — فلتر Butterworth صفري الطور (1-40 Hz):
- التردد المنخفض (1 Hz) بيشيل drift الكهربائي
- التردد العالي (40 Hz) بيشيل تلوث العضلات

**`sqi.py`** — مؤشر جودة كل قناة (σᵢ ∈ [0.1, 1.0]):
- بيفحص: ثبات التباين، flatline، تشبع ADC، عدم استقرار طيفي
- الحد الأدنى 0.1 (مش صفر) عشان مصفوفة التغاير ما تنهارش
- القنوات السيئة بتتخفض وزنها مش بتتشال — ده مهم عشان topology المصفوفة يفضل ثابت

**`covariance.py`** — 4 خطوات:
1. حساب مصفوفة التغاير الخام: `P = (1/(T-1)) XX^T`
2. ترجيح بـ SQI: `P_weighted = Σ^½ P Σ^½`
3. انكماش Ledoit-Wolf التلقائي: `P_reg = (1-λ)P + λI`
4. فرض SPD: تأريض القيم الذاتية عند `ε = 10⁻⁶`

### 3. التوجيه الذكي (`routing/`)

**`mode_router.py`** — بيحسب مقياسين ويقرر:

```
SQI_mean = متوسط جودة الـ 14 قناة
κ(P) = رقم الحالة (condition number) لمصفوفة التغاير

إذا SQI > 0.65 وَ κ < 50     →  RARD  (أعلى دقة)
إذا 0.35 < SQI ≤ 0.65 أو κ ≥ 50  →  MVES  (تدهور أنيق)
إذا القنوات السيئة > 70%       →  SAFE  (رفض)
```

### 4. استخراج الخصائص (`features/`)

**المسار الريماني — RARD (`rard_features.py`):**
- بيشتغل على مشعب SPD (Symmetric Positive Definite manifold)
- بيعمل إسقاط لمصفوفة التغاير على الفضاء المماسي عند نقطة مرجعية
- بينتج **105 خاصية** تمثل الاتصال المكاني بين القنوات الـ 14
- النقطة المرجعية بتتحسب بمتوسط Fréchet (المتوسط الهندسي على المشعب)

**المسار الإحصائي — MVES (`mves_features.py`):**
- **طاقة النطاقات** (Bandpower): Delta, Theta, Alpha, Beta — 56 خاصية
- **معاملات Hjorth**: Activity, Mobility, Complexity — 42 خاصية
- **الارتباط**: معاملات Pearson بين القنوات — 91 خاصية
- **الميل الطيفي**: لكل قناة — 14 خاصية
- **المجموع: 203 خاصية**

### 5. التدريب (`training/`)

**`train.py`** — التدريب بيستخدم:
- **GroupKFold**: تقسيم حسب الشخص — بيانات نفس الشخص أبداً ما تبقى في التدريب والاختبار في نفس الوقت (منع التسريب)
- **تشويش اصطناعي (30%)**: حقن ضوضاء، إسقاط قنوات، spikes — عشان الموديل يتعلم يتحمل الإشارات الوسخة
- **مصنفان**: LDA للـ RARD، Logistic Regression للـ MVES
- **مقاييس**: Accuracy, F1, Cohen's Kappa

### 6. محرك الاستدلال (`inference/`)

**`engine.py`** — الـ pipeline الكامل لكل نافذة EEG:
```
EEG خام → فلتر → SQI → تغاير → توجيه → خصائص → تصنيف → تنعيم زمني
```
- تنعيم EMA: `ŷ_t = 0.7 × y_t + 0.3 × ŷ_{t-1}`
- سياسة قبول/رفض النوافذ

**`personalization.py`** — تكيف مزدوج السرعة:
- **سريع (على الجهاز)**: EMA يوكليدي — `V = 0.95 V_prev + 0.05 V_new`
- **بطيء (على السيرفر)**: إعادة مركزة Fréchet على المشعب
- **حماية**: التكيف بيتجمد لو الإشارة سيئة (يمنع تلوث الـ baseline)

### 7. التصدير (`export/`)

**`export_onnx.py`** — تصدير للموبايل:
- تحويل sklearn models لـ ONNX format
- الموديل بيشتغل على Flutter عبر `onnxruntime_flutter`
- هدف الـ latency: < 35 مللي ثانية

## كيف تشغل المشروع

```bash
# 1. تفعيل البيئة
cd hazeclue-ai
source venv/bin/activate

# 2. حط بيانات STEW في data/raw/stew/
# 3. حط بيانات Mendeley في data/raw/mendeley/

# 4. شغل التدريب
python main.py \
  --stew-dir data/raw/stew \
  --mendeley-dir data/raw/mendeley \
  --folds 5 \
  --augment \
  --export-onnx

# 5. الموديلات هتتحفظ في trained_models/ و onnx_models/
```
